# AI 推广工作台规划文档

## 1. 当前阶段判断

项目当前处在本地 MVP 阶段：已经能把“输入推广需求 -> AI 生成图文 -> 生成图片 -> 填入发布表单 -> 调用 CLI 发布”跑成一条基础链路。

当前重点不是做成多人 SaaS，而是先把本机运营工作台做顺，验证内容生产、发布、数据采集和线索筛选是否能真正提升推广效率。

## 2. 已完成目标

### 2.1 基础工程

- 已搭建 Next.js 本地控制台项目。
- 已接入 HeroUI 作为页面组件库。
- 已形成 `/`、`/accounts`、`/generate` 三个基础页面。
- 已加入项目级设计文档 `PROJECT_DESIGN.md`。
- 已加入项目媒体生成规则：所有图片资产必须走本机 `gpt-image-2` workflow。

### 2.2 CLI 包装能力

- 已通过后端 API 启动本机子进程执行命令。
- 已支持调用 `social-auto-upload` 的 `sau` CLI。
- 已支持调用 `codex exec` 生成推广内容。
- 已优化子进程超时和退出处理，避免 CLI stdin 不关闭导致任务卡住。
- 已清洗命令输出中的 ANSI 控制符和终端字符画。
- 已保留原始 stdout/stderr，便于排查问题。

### 2.3 账号登录与状态

- 已支持小红书、抖音、快手账号登录入口。
- 已支持 Cookie 检查入口。
- 已新增“已登录平台”卡片，根据 `social-auto-upload/cookies` 判断本地账号状态。
- 已支持登录后刷新账号状态。

### 2.4 图文内容生成

- 已支持用户输入推广需求。
- 已支持 Codex CLI 输出结构化 JSON：
  - 标题。
  - 正文。
  - 标签。
  - 切入角度。
  - 图片需求。
  - 图片 prompt。
  - 安全提示。
- 已支持生成结果卡片展示标题、正文、标签、角度、安全提示。
- 已支持把生成内容填入发布表单。

### 2.5 图片生成

- 已接入本机 `gpt-image-2` 技能。
- 已在生成任务时检测 `gpt-image-2` Mode A。
- 已在 `image_required = true` 时直接调用 `scripts/generate.js` 生成图片。
- 已把生成图片保存到 `garden-gpt-image-2/image/`。
- 已把图片 prompt 保存到 `garden-gpt-image-2/prompt/`。
- 已提供受限图片预览接口 `/api/assets/image`。
- 已在页面展示真实图片预览、图片路径和 prompt 路径。
- 已把生成图片路径自动填入发布表单。

### 2.6 图文发布

- 已实现 `/api/publish/note`。
- 已支持小红书、抖音、快手图文发布。
- 已支持发布参数：
  - 平台。
  - 账号。
  - 标题。
  - 正文。
  - 标签。
  - 图片路径。
  - 定时发布。
  - 有界面/无头浏览器模式。
  - 抖音 BGM。
- 已加入发布前图片文件存在性校验。
- 已在 `/generate` 页面加入发布卡片，让生成和发布更接近同一工作流。

### 2.7 本地环境与安全

- 已支持项目本地 `.gateway.env` 配置图片生成网关。
- 已把 `.gateway.env`、`garden-gpt-image-2/`、`.next/`、`node_modules/` 等本地产物排除在 Git 提交之外。
- 已限制图片预览接口只能读取项目图片生成目录下的图片。

未完成内容：

## 4. 近期优先级建议

### P0：把发布闭环做稳

目标：确保“生成 -> 发布”在三个核心平台都能真实可用。

建议任务：

- 增加发布任务记录。
- 增加发布历史页面。
- 真实测试小红书、抖音、快手图文发布。
- 补平台限制校验。
- 捕获并展示更友好的平台异常。

### P1：补数据采集最小闭环

目标：发布后能知道哪些内容有效。

建议任务：

- 发布成功后记录作品 URL/ID。
- 做作品数据快照表。
- 先支持手动刷新数据，再做定时刷新。
- 做一个简单作品列表，展示点赞、收藏、评论、转发。
- 标记高表现内容。

### P2：做线索池

目标：从互动数据里筛选可跟进客户。

建议任务：

- 采集评论列表。
- 用 AI 判断评论意图。
- 建立线索表。
- 支持线索等级、标签、备注、跟进状态。
- 支持 CSV 导出。

### P3：提升内容生产效率

目标：从单条内容生成升级成批量推广生产。

建议任务：

- 批量生成 10-30 条选题。
- 每个选题生成多个平台版本。
- 根据历史数据反推下次内容角度。
- 增加内容模板库。
- 增加敏感词和风险表达检查。

## 5. 发展方向

### 5.1 本地运营控制台

继续保持本机运行，不做复杂账号系统。

适合阶段：

- 个人运营。
- 小团队内部工具。
- 账号 Cookie 和浏览器会话都在本机管理。

优势：

- 实现快。
- 平台登录和浏览器自动化更容易。
- 不需要处理云端账号托管风险。

风险：

- 不适合多人协作。
- 长任务依赖本机在线。
- 数据备份需要额外设计。

### 5.2 半自动推广工作流

让系统负责重复劳动，人负责审核和关键决策。

推荐形态：

- AI 批量生成内容。
- 人工审核。
- 系统自动发布。
- 系统自动采集数据。
- AI 辅助筛选线索。
- 人工跟进客户。

这个方向比较稳，不容易因为完全自动发布导致内容质量和平台风险失控。

### 5.3 数据驱动内容优化

把发布数据反馈给内容生成模块。

可做能力：

- 识别高收藏内容。
- 识别高评论内容。
- 识别高转化线索内容。
- 总结有效标题结构。
- 总结有效封面风格。
- 自动生成下一轮内容策略。

### 5.4 CRM 轻量化

不急着做完整 CRM，先做推广线索池。

可做能力：

- 用户去重。
- 线索等级。
- 线索标签。
- 跟进状态。
- 备注。
- 导出。
- 私信话术生成。

### 5.5 视频内容链路

图文跑通后，可以扩展视频。

可做能力：

- 口播脚本生成。
- 分镜生成。
- 封面生成。
- 视频合成。
- 多平台视频发布。
- 视频数据采集。

## 6. 当前技术风险

- 平台页面和发布流程可能经常变化，`social-auto-upload` 需要持续维护。
- 平台登录、验证码、风控会影响自动发布稳定性。
- 生成图片和生成文案都依赖外部 API，可能遇到超时、限额、费用和鉴权问题。
- 当前没有数据库，任务历史和数据分析能力无法沉淀。
- 当前没有队列系统，长任务并发和取消能力弱。
- 自动化发布涉及平台规则，需要避免违规、刷屏和误导性内容。

- 增加任务历史页面。

### Milestone 2：作品数据采集

- 记录作品 URL/ID。
- 手动刷新作品数据。
- 定时刷新作品数据。
- 增加作品表现列表。
- 增加基础趋势对比。

### Milestone 3：线索管理

- 采集评论和互动用户。
- AI 识别意图。
- 建立线索列表。
- 支持筛选和导出。

### Milestone 4：批量内容生产

- 批量生成选题。
- 批量生成图文。
- 支持多平台差异化版本。
- 根据数据反馈优化下一批内容。
